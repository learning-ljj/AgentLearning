## 1. 基础定义
- **JSON**：JavaScript 对象表示法的纯文本数据格式，完全独立于语言。  
- **文件后缀**：`.json`  
- **MIME 类型**：`application/json`

## 2. 仅两种合法容器  
| 容器 | 符号 | 内部存放内容 | 示例 |
|------|------|--------------|------|
| **对象** | `{ }` | 无序的 **键值对** 集合 | `{ "name": "张三", "age": 20 }` |
| **数组** | `[ ]` | 有序的 **值** 列表 | `[ "苹果", 123, true, { }, [ ] ]` |

## 3. JSON 语法规则

**键值对语法**  
- 格式：`"键名": 值`  
- 键名**必须**用双引号包裹。  
- 示例：`"price": 19.99`

**值的合法种类**  
| 类型 | 写法示例 | 注意 |
|------|----------|------|
| 字符串 | `"hello"` | **必须用双引号**，单引号非法 |
| 数字 | `123`, `3.14`, `-5`, `1.2e10` | 不带引号，禁止前导零（如 `01`） |
| 布尔值 | `true`, `false` | 小写，不带引号 |
| 空值 | `null` | 小写，不带引号 |
| 对象 | `{ "key": "value" }` | 可嵌套 |
| 数组 | `[ 1, 2, 3 ]` | 可嵌套 |

## 4. 注意事项

**空白与格式**  
- 空格、换行、缩进完全自由（不影响解析），仅用于美化可读性。

**不可变红线**  
- ❌ 不能在数组里直接写 `"键": 值`（键值对必须包在 `{}` 内）。  
- ❌ 字符串不能用单引号 `'text'`。  
- ❌ 最后一个元素后面**不能**加逗号（严格解析器会报错）。  
- ❌ 不支持注释（`//` 或 `/* */` 均非法）。  
- ❌ 数字不能是八进制（如 `012`）、不能是 `NaN` 或 `Infinity`。

**常见错误对照**  
| 错误写法 | 正确写法 | 解释 |
|----------|----------|------|
| `[ "a": 1 ]` | `[ { "a": 1 } ]` 或 `{ "a": 1 }` | 数组内不能直接放键值对 |
| `{ name: "tom" }` | `{ "name": "tom" }` | 键名必须加双引号 |
| `{ "a": 1, }` | `{ "a": 1 }` | 结尾逗号禁止 |
| `[ 'text' ]` | `[ "text" ]` | 字符串只能用双引号 |
